# 个人收支记账系统使用说明书

## 1. 系统简介

个人收支记账系统是一个基于 Flask + SQLite 的本地 Web 应用，用于记录日常收入和支出。系统支持注册登录、手动记账、AI 自然语言识别记账、记录筛选、删除记录、CSV 导出、基础统计和月度智能分析。

## 2. 运行环境

- Windows 10/11
- Python 3.12 或兼容版本
- 浏览器：Chrome、Edge 或 Codex 内置浏览器

项目依赖在 `requirements.txt` 中。当前 AI 调用使用 Python 标准库完成，不需要额外安装 OpenAI SDK。

## 3. 启动系统

进入项目目录：

```powershell
cd D:\shouzhixitong\Finance-Tracker
```

创建虚拟环境：

```powershell
python -m venv .venv
```

安装依赖：

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

启动系统：

```powershell
.\.venv\Scripts\python.exe app.py
```

浏览器打开：

```text
http://127.0.0.1:5000/login
```

## 4. DeepSeek API 配置

AI 记账助手和月度智能分析通过 DeepSeek 的 OpenAI 兼容 Chat Completions 接口调用模型。配置全部使用环境变量，不要把密钥写入代码。

必填：

```powershell
$env:DEEPSEEK_API_KEY="你的 DeepSeek API 密钥"
```

可选：

```powershell
$env:DEEPSEEK_BASE_URL="https://api.deepseek.com"
$env:DEEPSEEK_MODEL="deepseek-v4-flash"
$env:DEEPSEEK_TIMEOUT="20"
```

如果没有配置 `DEEPSEEK_API_KEY`，AI 功能会显示中文错误提示，手动记账、筛选和统计功能仍可正常使用。系统也兼容旧环境变量 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`、`LLM_TIMEOUT`，但建议新配置统一使用 `DEEPSEEK_*`。

## 5. 注册与登录

1. 在登录页点击“立即注册”。
2. 输入用户名、邮箱、手机号和密码。
3. 注册成功后返回登录页。
4. 输入用户名和密码后点击“登录”。

注意：当前项目用于课程练习，密码仍以明文方式存储在本地 SQLite 中，不建议存放真实敏感信息。

## 6. 首页

登录后首页展示：

- 收入总额
- 支出总额
- 当前结余
- 本月每日支出折线图
- 月度支出趋势折线图

金额统一使用人民币符号 `¥`。

## 7. 手动新增交易

1. 点击导航栏“交易记录”。
2. 点击“新增交易”。
3. 填写类型、日期、分类、金额、支付方式和备注。
4. 点击“保存交易”。

系统会校验：

- 类型必须是“收入”或“支出”。
- 金额必须大于 0。
- 分类必须与类型匹配。
- 日期必须有效。
- 支付方式必须有效。

## 8. AI 自然语言记账

在“交易记录”页面顶部的“AI 记账助手”输入框中输入自然语言，例如：

```text
今天午餐花了28元
昨天工资收入5000元
打车18.5元
```

点击“AI 识别”后，系统会调用模型并尝试识别：

- 日期
- 金额
- 收入/支出类型
- 分类
- 支付方式
- 备注

识别结果不会直接入库，而是自动填入“新增交易”弹窗。请确认或修改后再点击“保存交易”。

如果模型未配置、超时、返回空内容、JSON 格式错误或金额无法识别，页面会显示错误提示，并保留手动表单供继续记账。

## 10. 筛选交易记录

“交易记录”页面支持按以下条件筛选：

- 类型：收入、支出或全部
- 分类
- 开始日期
- 结束日期

点击“筛选”后，列表只展示符合条件的记录。点击“重置”可恢复全部记录。

## 11. 删除记录

在交易记录表格中，点击对应行右侧的删除按钮即可删除该记录。系统会限定只能删除当前登录用户自己的记录。

## 12. 导出 CSV

在“交易记录”页面点击“导出 CSV”，会导出当前列表中的记录。

导出文件名：

```text
交易记录.csv
```

CSV 表头：

```text
日期,类型,分类,金额,支付方式,备注
```

## 13. 统计分析

点击导航栏“统计分析”，可以查看：

- 收入总额
- 支出总额
- 当前结余
- 按类型和分类统计金额
- 支出最高的分类

点击“生成月度消费分析”后，系统会基于当前登录用户本月记录生成：

- 月度摘要
- 主要支出分类
- 节省建议

如果本月没有记录，系统会提示先新增记录。

## 14. 数据存储

系统使用 SQLite 保存数据，数据库文件位于项目根目录：

```text
finance_tracker.db
```

首次启动会自动建表。旧版本数据库启动后会自动给 `transactions` 表补充 `type` 字段，旧记录默认作为支出处理。

## 15. 常见问题

### 页面打不开

确认 Flask 服务正在运行，并访问：

```text
http://127.0.0.1:5000/login
```

### AI 识别失败

检查是否配置了 `DEEPSEEK_API_KEY`，以及 `DEEPSEEK_BASE_URL` 和 `DEEPSEEK_MODEL` 是否匹配你的 DeepSeek 模型服务。

### 筛选不到数据

确认筛选条件是否过窄。可以点击“重置”查看全部记录。

### 图表没有数据

首页图表只统计支出记录。请先新增至少一条支出记录。

### 中文显示异常

确认文件使用 UTF-8 编码保存，并使用支持中文的浏览器访问。

## 16. 使用建议

- 定期备份 `finance_tracker.db`。
- 课程练习版本不要直接部署到公网处理真实敏感财务数据。
- 如果要正式使用，建议继续补充密码哈希、CSRF 防护、生产部署配置和更完善的权限校验。
